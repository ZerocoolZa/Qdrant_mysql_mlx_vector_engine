# Config Cascade Pipeline — Extract → Generate → Verify → All Code Reads from Config

> **Core thesis:** No hardcoded values. Every path, port, timeout, DB path,
> threshold, and constant lives in a `Config.py` file. ConfigCascade scans `.py`
> files, extracts config-like data, generates/updates config files, and verifies
> compliance — so all code reads from config instead of using hardcoded values.
>
> **The chain:** .py files (with hardcoded values) → SCAN → EXTRACT →
> GENERATE Config.py → VERIFY → all code imports from Config → no hardcode.

---

## Pipeline Overview

```
.py Files (with hardcoded values)
       ↓
  SCAN — walk folder, find all .py files, detect UPPERCASE constants
       ↓
  EXTRACT — AST-parse each file, pull constants, dicts, paths, defaults
       ↓
  GENERATE — create Config.py for domain (VBStyle headers + extracted data)
       ↓
  WRITE — write new Config.py or overwrite existing
       ↓
  UPDATE — update specific keys in existing Config.py
       ↓
  VERIFY — check VBStyle compliance (headers, Run(), Tuple3, no print, no hardcode)
       ↓
  CATALOG — list all Config.py files across project, compliance status
       ↓
  RESULT — all code imports from Config → no hardcoded values
```

## Stages

### Stage 1: SCAN — Walk Folder, Find Config-Like Constants

**Tool:** `ConfigCascade.Run("scan", {"path": "..."})`

Walks a directory tree, finds all `.py` files, and for each file:
- AST-parses the source
- Detects module-level `UPPERCASE = value` assignments
- Detects whether file has a `Config` class
- Detects whether file has a `Run()` method
- Returns per-file: constants, constant_count, has_config_class, has_run_method

**What gets detected as config-like:**
- `PORT = 8080` — integer constant
- `DB_PATH = os.path.join(BASE_DIR, "data.db")` — path expression
- `TIMEOUT = 30` — numeric constant
- `HOST = "localhost"` — string constant
- `MAX_RETRIES = 3` — integer constant
- `COLORS = {"bg": "#0d1117", "text": "#e6edf3"}` — dict constant
- `FILES = ["a.py", "b.py"]` — list constant

### Stage 2: EXTRACT — Pull Config Data from a Single File

**Tool:** `ConfigCascade.Run("extract", {"file": "..."})`

AST-parses a single `.py` file and extracts:

| What | How | Example |
|---|---|---|
| Module-level constants | `ast.Assign` with `target.id.isupper()` | `PORT = 8080` |
| String constants | `ast.Constant` value | `HOST = "localhost"` |
| Numeric constants | `ast.Constant` value | `TIMEOUT = 30` |
| List constants | `ast.List` → recursive element extraction | `FILES = ["a.py"]` |
| Dict constants | `ast.Dict` → recursive key/value extraction | `COLORS = {"bg": "#fff"}` |
| Path expressions | `ast.BinOp` with `ast.Add` (string concat) | `os.path.join(BASE_DIR, "x")` |
| Function calls | `ast.Call` → `func_name(args)` | `os.path.dirname(__file__)` |
| Class names | `ast.ClassDef` | `class Config:` |
| Run() method | `ast.FunctionDef` named "Run" inside class | `def Run(self, ...)` |

**AST value resolver (`_ast_to_value`):**
- `ast.Constant` → `node.value`
- `ast.List` → `[element_values]`
- `ast.Tuple` → `(element_values)`
- `ast.Dict` → `{key_values: value_values}`
- `ast.Name` → `node.id` (variable reference)
- `ast.Attribute` → `"value.attr"` (e.g. `os.path`)
- `ast.BinOp(Add)` → string concatenation if both sides are strings
- `ast.Call` → `"func_name(arg1, arg2)"`

### Stage 3: GENERATE — Create Config.py for a Domain

**Tool:** `ConfigCascade.Run("generate", {"path": "...", "domain": "..."})`

For a domain folder that doesn't have a `Config.py`:

1. **Scan** the folder for all constants across all `.py` files
2. **Deduplicate** — first occurrence wins, skip `__` prefixed names
3. **Build config content** with:
   - BCL headers (`[@GHOST]`, `[@VBSTYLE]`, `[@SUMMARY]`, `[@CLASS]`, `[@METHOD]`)
   - `import os`
   - `BASE_DIR = os.path.dirname(os.path.abspath(__file__))`
   - `DOMAIN = "domain_name"`
   - `VERSION = "1.0"`
   - All extracted constants as `UPPERCASE = value`
   - `class Config:` with `__init__`, `Run("get"/"set")`, `read_state`, `set_config`
4. **Write** to `Config.py` in the target folder

**Generated Config.py structure:**
```python
# [@GHOST]{[@file<Config.py>][@domain<DomainName>][@role<config>][@auth<cascade>][@date<2026-06-27>][@ver<1.0>]}
# [@VBSTYLE]{[@auth<cascade>][@role<config>][@return<Tuple3>][@orch<none>][@no<decorators|print|hardcoded|tabs|self_underscore>][@model<one_class_one_domain_one_authority_complete>]}
# [@SUMMARY]{Config for DomainName — generated by ConfigCascade}

BASE_DIR = os.path.dirname(os.path.abspath(__file__))
DOMAIN = "DomainName"
VERSION = "1.0"
```

### Stage 4: READ — Load Existing Config

**Tool:** `ConfigCascade.Run("read", {"file": "..."})`

Same as extract but for an existing `Config.py` — returns all constants,
class names, and compliance indicators.

### Stage 5: WRITE — Write Config from Scratch

**Tool:** `ConfigCascade.Run("write", {"path": "...", "domain": "...", "constants": {...}})`

Write a new `Config.py` with user-supplied constants (not extracted from files).

### Stage 6: UPDATE — Update Specific Keys

**Tool:** `ConfigCascade.Run("update", {"file": "...", "keys": {"DB_PATH": "/new/path.db"}})`

For each key:
- If key exists in file → regex replace `^KEY = .*$` with `KEY = new_value`
- If key doesn't exist → insert at top of file

### Stage 7: VERIFY — VBStyle Compliance Check

**Tool:** `ConfigCascade.Run("verify", {"file": "..."})`

Checks 11 compliance rules:

| Check | What It Tests |
|---|---|
| `has_ghost_header` | `[@GHOST]` present in source |
| `has_vbstyle_header` | `[@VBSTYLE]` present in source |
| `has_summary` | `[@SUMMARY]` present in source |
| `has_class_header` | `[@CLASS]` present in source |
| `has_run_method` | `def Run(self,` present |
| `has_tuple3_return` | `(1,` and `None)` present |
| `no_print` | `print(` not in source |
| `no_decorators` | `@property`, `@staticmethod`, `@classmethod` not in source |
| `no_self_underscore` | `self._` not in source |
| `no_tabs` | `\t` not in source |
| `has_state_dict` | `self.state` in source |

Returns: `compliant` (bool), `passed` (count), `failed` (list of check names), `checks` (full dict)

### Stage 8: CATALOG — List All Config Files in Project

**Tool:** `ConfigCascade.Run("catalog", {"root": "..."})`

Walks the entire project tree and finds all config files:
- `config.py`, `Config.py`, `Config_*.py`, `config_*.py`

For each config file:
- Verify VBStyle compliance
- Extract constants
- Record: file path, domain, compliant, constant_count, has_config_class, has_run_method

Returns: total_config_files, compliant count, non-compliant count, per-file details

## Companion: ConfigExtractor (Regex-Based)

**Tool:** `gui_engine/config_extractor.py`

A lightweight regex-based extractor that works even on files with syntax errors
(no AST needed):

- Module-level constants: `UPPERCASE = value`
- String literals in code
- Numeric literals in code
- Default parameters: `def foo(self, x="value", y=42)`
- `get()` fallbacks: `config.get("key", "fallback")`
- Class names and method names

**Usage:** `python3 config_extractor.py <source.py> > Config.py`
