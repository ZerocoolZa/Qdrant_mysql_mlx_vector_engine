# BND — Best Next Direction (the evolution from question to direction)

## The shift

BNQ was interrogative — "what should I ask next?"
BND is directional — "where should the system move next?"

The primitive changed:
- question → direction
- language-based → graph-based
- conversational → structural
- subjective → score-based

## What BND is

BND is not a question. It is a directional edge in a semantic graph that maximizes forward progress.

```
BND = argmax(edge_weight(node -> next_node))
```

BND = the highest-weight transition from the current knowledge state to the next most informative state in the idea graph.

## Direction types

| Direction | Meaning |
|---|---|
| Expansion | Go deeper into the concept |
| Decomposition | Split into parts/classes/modules |
| Constraint tightening | Add rules, limits, or structure |
| Implementation shift | Move from design → code |
| Validation shift | Move from build → test |

## BNQ vs BND in system terms

| Type | Role |
|---|---|
| BNQ | Branching function — expands state space (discovery) |
| BND | Path selection function — moves along optimal path (traversal) |

BNQ expansion = BFS/DFS hybrid. BND selection = heuristic A*.

This is not conversation logic. This is a search algorithm over conceptual space.

## The full state machine

```
STATE MACHINE
State S0 = seed
Graph G = {}
Frontier F = {S0}

LOOP
Step 1 — Expand:   C = BNQ(S) ∪ BND(S)
Step 2 — Score:    score(c) = information_gain + dependency_unlock + structural_progress + execution_closeness
Step 3 — Select:   next = argmax(score(C))
Step 4 — Transition: S <- next, add edge(S_old -> S)
Step 5 — Stop:     if max(score(C)) < threshold → CLOSURE
```

Simplified:
```
while frontier_not_empty:
    candidates = expand(BNQ + BND)
    best = score(candidates)
    move(best)
```

## Termination = closure

When no BNQs or BNDs remain:
- Graph is saturated
- No new information paths exist
- All transitions are low-value
- System reaches CLOSURE STATE = complete knowledge expansion

## One-line definition

BNQ/BND is a dual-operator graph traversal system that converts a seed concept into a fully explored and exhausted knowledge structure through scored expansion and directional movement.

## The key relationship: BND doesn't need BNQ

**Step one is cluster.** Give a word. Cluster it out — all angles.

At the end of clustering, best next question is answered by best next direction. BND IS the answer to BNQ.

**Therefore BND does not require BNQ to proceed.** The direction IS the answer. You can skip the question and go straight to direction.

- BNQ is optional
- BND is the primitive
- The question was just the old UI layer
- The direction is the engine

## Scoring formula

```
score(c) = information_gain + dependency_unlock + structural_progress + execution_closeness
```

- information_gain: how much new knowledge this move adds
- dependency_unlock: how many blocked paths this opens
- structural_progress: how much closer to complete architecture
- execution_closeness: how much closer to runnable code

# R = Restriction Layer (policy-gated graph traversal)

## The shift

The system is not just expand + score + move. It is **expand + score + RESTRICT + then move**.

```
valid_actions = filter(BNQ ∪ BND, R)
next = argmax(valid_actions)
```

## What restriction means

Restriction is not limitation. It is **structure enforcement**.

| Restriction type | What it enforces |
|---|---|
| Structural | Must reduce ambiguity, increase resolution, move toward implementability |
| Domain | Stay within current concept space, no external drift unless allowed |
| Progress | BNQ allowed only if unresolved nodes exist, BND allowed only if a path is viable |
| Saturation | Prevent infinite branching, prune low-value edges |

## The key insight

**Intelligence is not choosing the best next step. Intelligence is deciding which TYPE of step is even valid.**

That is a higher-level control system. R is a policy engine, not a filter. It is a governor over the idea-space machine.

## Control loop with restriction

```
while not terminated:
    candidates = BNQ(S) + BND(S)
    allowed = R.filter(candidates)
    if allowed empty:
        stop
    next = argmax(score(allowed))
    S = next
```

## One-line definition

BNQ/BND with restriction is a policy-gated graph traversal system where expansion and movement operators are dynamically enabled or disabled based on structural, semantic, and progression constraints.
